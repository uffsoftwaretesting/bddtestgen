User Story 10: Change Account Password
Description:
As a user, I want to change my current password to a new one to keep my account secure.
Screen Title: Change Password
The form must contain the following fields:
Current Password (inputPassword) *
New Password (inputPassword) *
Confirm New Password (inputPassword) *
Buttons:
UPDATE PASSWORD
When the UPDATE PASSWORD button is clicked, the system validates the form. If successful, the password is changed, the user is logged out for security, and redirected to the login page with a message: "Your password has been changed successfully. Please log in again."
Business Rules:
BR01 - All fields are mandatory.
Error message: "This field is required!"
BR02 - The Current Password must match the user's existing password.
Error message: "Incorrect current password."
BR03 - The New Password must meet security requirements (e.g., 8+ characters, one uppercase, one number).
Error message: "Password must be at least 8 characters long and include an uppercase letter and a number."
BR04 - The New Password and Confirm New Password fields must match.
Error message: "The new passwords do not match."
BR05 - The New Password cannot be the same as the Current Password.
Error message: "Your new password cannot be the same as your old one."
